# Account Setup

Prepare accounts before launch week. Send all launch traffic to the GitHub repo:

https://github.com/kaibuild/backupproof

## Hacker News

- Create an account ahead of launch.
- Use a Show HN post.
- Link directly to the GitHub repo.
- Keep the post concise and technical.
- Be available for questions after posting.

## Reddit

- Create the account ahead of launch.
- Comment naturally in r/selfhosted.
- Do not post the product yet.
- Do not paste links during warm-up.
- Build some karma.
- Target 2-4 weeks of normal participation if possible.
- When ready, post the prepared r/selfhosted draft and link to GitHub.

## Lemmy

- Create an account on an instance with active self-hosting communities.
- Read community rules before posting.
- Keep the post shorter than the Reddit version.
- Link to GitHub.

## Dev.to

- Create or update the author profile.
- Publish the article draft during launch week.
- Link to GitHub near the end.
- Respond to technical comments.

## Medium

- Create or update the author profile.
- Adapt the article for a broader technical audience.
- Link to GitHub.

## Hackernoon

- Create or update the author profile.
- Adapt the article for a practical engineering audience.
- Link to GitHub.

## X

- Prepare a short post and one follow-up reply with the GitHub link.
- Keep it direct and technical.
- Do not overstate what the MVP does.

## LinkedIn

- Use the prepared technical post.
- Avoid launch hype.
- Link to GitHub.

## Newsletter

- Prepare a short note:
  - what BackupProof is
  - who it is for
  - what it checks
  - what it does not do
  - GitHub link

## Platform Timing

- Launch in the same week.
- Send traffic to GitHub, not to a separate landing page.
- Keep tone humble, technical, transparent, and useful.
