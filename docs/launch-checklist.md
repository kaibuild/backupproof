# Launch Checklist

## GitHub Readiness

- README explains what BackupProof is in the first screen.
- Quick Start works from a fresh clone.
- Docker build and Docker run examples work.
- Example report is committed.
- LICENSE, CONTRIBUTING, SECURITY, and issue templates are present.
- Roadmap issues are created or documented.
- Repository description points to the core one-liner.
- No secrets, `.env` files, `node_modules`, `dist`, build artifacts, or coverage output are committed.

## Reddit Account Readiness

- Create the Reddit account ahead of launch.
- Comment naturally in r/selfhosted.
- Do not post the product during warm-up.
- Do not paste links during warm-up.
- Build some karma.
- Target 2-4 weeks of normal participation if possible.

## Hacker News Readiness

- Create the Hacker News account ahead of launch.
- Use a clear Show HN title.
- Link directly to the GitHub repo.
- Keep the post concise and technical.
- Stay around to answer questions.

## Lemmy Readiness

- Pick the most relevant self-hosting community.
- Read community rules before posting.
- Keep the post short and practical.
- Link to GitHub.

## Dev.to / Medium / Hackernoon Article Checklist

- Lead with the backup readiness problem, not the product.
- Explain common Compose backup mistakes.
- Include concrete examples.
- Link to the GitHub repo.
- Make limitations explicit.
- Avoid corporate launch language.

## Launch-Week Schedule

- Day 1: publish GitHub repo and verify README, examples, and Docker commands.
- Day 2: post Show HN.
- Day 3: post to r/selfhosted if the account is ready.
- Day 4: post to Lemmy.
- Day 5: publish Dev.to article.
- Day 6: publish Medium or Hackernoon adaptation.
- Day 7: summarize feedback and open follow-up issues.

## Post-Launch Response Checklist

- Reply to bug reports with reproduction steps.
- Ask for minimal Compose snippets without secrets.
- Label issues consistently.
- Convert repeated feedback into roadmap issues.
- Thank contributors without overpromising timelines.
- Keep all traffic pointed at the GitHub repo.
